# SuperRead — AI 驱动的资讯简报工具

> 订阅你关注的信息源。每天 AI 自动汇总摘要。用5分钟替代2小时。

## 为什么需要 SuperRead？

关注技术新闻、行业博客和开发者动态意味着你要刷几十个网站——或者被 RSS 阅读器里成百上千篇未读文章淹没。你不需要每篇文章的全文，你只需要知道今天发生了什么。

**SuperRead** 订阅你指定的信息源，定时抓取新内容，用 AI 将一切浓缩为一份每日简报。花几分钟看摘要，感兴趣的再点原文链接。

## 功能

| 功能 | 说明 |
|------|------|
| **订阅源管理** | 添加 RSS 源、自动发现网站订阅、导入 OPML 文件 |
| **定时抓取** | 后台每30分钟自动拉取最新内容 |
| **AI 摘要生成** | 每篇文章浓缩为一句话摘要 |
| **每日简报** | 当日所有来源的更新，一份滚动页面看完 |
| **智能去重** | 同一事件被多个来源报道？合并为一条 |
| **阅读管理** | 已读/未读、收藏星标、标签分类、稍后阅读 |
| **通知推送** | 平台内通知 + 可选邮件推送 |
| **原文链接** | 每条摘要附带原文链接 |

## 多用户支持

面向所有平台用户，每人独立订阅和简报。纯阅读工具，无社交功能。

## 技术栈

| 环境 | 后端 | 数据库 | 前端 | 特殊依赖 |
|------|------|--------|------|---------|
| 在线版（Oxelia51） | Go + cron | PostgreSQL | React | RSS 解析 + LLM API |
| 桌面版（exe） | Go + cron | SQLite | 内嵌 React | 同上 |

- 定时任务使用 Go 内置 cron，无需外部调度器
- AI 摘要需自带 API Key，成本自己控制

## API Key

SuperRead 使用你自己的大模型 API Key 完成摘要。Key 存储在本地。你可以通过调整抓取频率和摘要长度来控制费用。

## 使用方式

### 在线版

1. 访问 [oxelia51.com](https://oxelia51.com) 注册登录
2. 从工具菜单打开 SuperRead
3. 添加 RSS 来源，填入 API Key
4. 每天查看你的简报

### 桌面版（exe）

1. 从 [GitHub Releases](https://github.com/XiaoleC05/SuperRead/releases) 下载 `SuperRead.exe`
2. 运行即启动本地界面
3. 添加来源和 API Key，一切在本地运行
