# SuperRead — AI 资讯简报

> 关注什么，订什么。每天自动帮你总结好，五分钟看完今日大事。

## 痛点在哪

想看技术新闻、行业动态、技术博客，要么刷几十个网站，要么在 RSS 阅读器里攒几百条未读然后干脆不看。你其实不需要每篇的全文——你只想知道今天有什么值得关注的。

**SuperRead** 帮你盯住你指定的信息源，定时去抓，抓回来让 AI 帮你总结成一份每日简报。看摘要就几分钟，感兴趣的再点原文。

## 能做什么

| 功能 | 说明 |
|------|------|
| **信息源管理** | 贴 RSS 链接就能订阅，输了网址能自动发现，还支持 OPML 一键导入 |
| **定时抓取** | 后台每半小时自动跑一次，新内容不遗漏 |
| **AI 总结** | 每篇文章浓缩成一句话，说清楚核心讲什么 |
| **每日简报** | 所有来源的今日更新汇总在一个页面，一滚到底看完 |
| **自动去重** | 同一件事好几家都报了？合为一条，不重复刷屏 |
| **阅读管理** | 已读/未读、加星收藏、打标签、稍后看 |
| **消息通知** | 平台内弹提醒，也可以开邮件推送 |
| **原文直达** | 每条摘要后面都带原文链接 |

## 支持多用户

平台所有用户都能用，每人独立订阅、独立简报。就是个纯阅读工具，没有关注评论那一套。

## 技术选型

| 环境 | 后端 | 数据库 | 前端 | 特别依赖 |
|------|------|--------|------|---------|
| 线上（Oxelia51） | Go + cron | PostgreSQL | React | RSS 解析 + 大模型 API |
| 桌面（exe） | Go + cron | SQLite | 内嵌 React | 同上 |

- 定时任务用 Go 自带的能力，不加额外东西
- AI 总结要用你自己的 API Key，花多少钱你自己控制

## 费用说明

SuperRead 做 AI 总结需要你提供自己的大模型 API Key。Key 只在你本地存着。想省钱就调低抓取频率，或缩短摘要长度。

## 怎么用

### 线上

1. 打开 [oxelia51.com](https://oxelia51.com) 注册登录
2. 工具菜单里找到 SuperRead
3. 加上你想看的 RSS 来源，填上 API Key
4. 每天回来扫一眼简报

### 桌面版

1. 去 [GitHub Releases](https://github.com/XiaoleC05/SuperRead/releases) 下载 `SuperRead.exe`
2. 双击运行
3. 加来源、填 Key，全在你电脑上跑
